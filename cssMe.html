Introduction to CSS

CSS (Cascading Style Sheets) describes how HTML elements are displayed. You can use inline, internal, or external CSS. Syntax: selector { property: value; }
Example 1: Inline CSS
<p style="color: red;">Hello!</p>
Output: Hello!
Example 2: Internal CSS
<style>p { color: blue; }</style>
Output: Hello!
Example 3: External CSS
<link rel="stylesheet" href="style.css">
Output: Loads and applies style.css.
Example 4: Basic CSS Syntax
h1 { color: green; }
Output: Heading
Example 5: Multiple Selectors
h1, h2 { font-family: Arial; }
Output: H1/H2 text
Example 6: Class Selector
.highlight { background: yellow; }
Output: Highlighted
Example 7: ID Selector
#main { font-size: 2em; }
Output: Main
Example 8: Universal Selector
* { margin: 0; }
Output: All elements have no margin.
Example 9: Comments in CSS
/* This is a comment */
Output: No visible output.
Example 10: Case Insensitivity
BODY { background: #eee; }
Output: Body background is light gray.
Example 11: Property Value Pair
p { color: purple; }
Output: Purple text
Example 12: Multiple Declarations
h2 { color: orange; font-weight: bold; }
Output: Orange Bold
Example 13: CSS File Extension
style.css
Output: External CSS files use the .css extension.
Example 14: Importing CSS
@import url('style.css');
Output: Imports another CSS file.
Example 15: CSS Specificity
#id .class p { color: teal; }
Output: Most specific selector wins.
Example 16: Inheritance
body { color: navy; }
Output: All text inherits navy color unless overridden.
Example 17: !important Rule
p { color: red !important; }
Output: Red text (forced)
Example 18: Linking Multiple Stylesheets
<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="theme.css">
Output: Both CSS files are loaded.
Example 19: Overriding Styles
p { color: blue; } p { color: green; }
Output: Green text
Example 20: CSS for Print
@media print { body { color: black; } }
Output: Text is black when printing.

Media Queries & Responsive Design Basics

Media queries allow you to apply CSS rules based on device characteristics, enabling responsive layouts for different screen sizes.
Example 1: Basic Media Query
@media (max-width: 600px) { body { background: #f0f; } }
Output: Background turns purple on small screens.
Example 2: Responsive Font Size
@media (max-width: 500px) { h1 { font-size: 1.2em; } }
Output: Heading font size shrinks on mobile.
Example 3: Hide Element on Mobile
@media (max-width: 600px) { .sidebar { display: none; } }
Output: Sidebar hidden on small screens.
Example 4: Change Layout on Tablet
@media (min-width: 601px) and (max-width: 900px) { .container { flex-direction: column; } }
Output: Container stacks vertically on tablets.
Example 5: Responsive Image
img { max-width: 100%; height: auto; }
Output: Image scales to fit container.
Example 6: Responsive Navigation
@media (max-width: 700px) { nav { flex-direction: column; } }
Output: Navigation stacks vertically on small screens.
Example 7: Responsive Padding
@media (max-width: 400px) { .box { padding: 5px; } }
Output: Less padding on very small screens.
Example 8: Responsive Columns
@media (max-width: 800px) { .col { width: 100%; } }
Output: Columns become full width on mobile.
Example 9: Responsive Button Size
@media (max-width: 500px) { button { font-size: 1.1em; } }
Output: Buttons are easier to tap on mobile.
Example 10: Responsive Grid Gap
@media (max-width: 600px) { .grid { gap: 8px; } }
Output: Grid gap shrinks on small screens.
Example 11: Responsive Flex Direction
@media (max-width: 600px) { .flex { flex-direction: column; } }
Output: Flex items stack vertically on mobile.
Example 12: Responsive Table
@media (max-width: 700px) { table { font-size: 0.9em; } }
Output: Table text shrinks on small screens.
Example 13: Responsive Card Layout
@media (max-width: 600px) { .card { margin: 8px 0; } }
Output: Cards have more vertical margin on mobile.
Example 14: Responsive Hide Text
@media (max-width: 400px) { .long-text { display: none; } }
Output: Long text hidden on very small screens.
Example 15: Responsive Border Radius
@media (max-width: 500px) { .box { border-radius: 0; } }
Output: No border radius on mobile.
Example 16: Responsive Font Weight
@media (max-width: 600px) { h2 { font-weight: 400; } }
Output: Headings are lighter on mobile.
Example 17: Responsive List Style
@media (max-width: 600px) { ul { list-style-type: none; } }

Example 18: Responsive Image Border
@media (max-width: 500px) { img { border-radius: 50%; } }

Example 20: Responsive Navbar

Advanced Selectors & Combinators

Advanced selectors and combinators allow you to target elements more precisely using attributes, relationships, pseudo-classes, and pseudo-elements.
Example 1: Attribute Selector (exact match)
input[type="text"] { border: 2px solid #007bff; }
Output: Text inputs have a blue border.
Example 2: Attribute Selector (starts with)
a[href^="https"] { color: green; }
Output: Links starting with https are green.
Example 3: Attribute Selector (ends with)
img[src$=".png"] { border: 1px solid #f44336; }
Output: PNG images have a red border.
Example 4: Attribute Selector (contains)
input[placeholder*="name"] { background: #e3f2fd; }
Output: Inputs with "name" in placeholder have blue background.
Example 5: Child Selector
ul > li { color: #ff9800; }
Output: Direct li children of ul are orange.
Example 6: Descendant Selector
div span { color: #4caf50; }
Output: Spans inside divs are green.
Example 7: Adjacent Sibling Selector
h2 + p { font-style: italic; }
Output: Paragraphs immediately after h2 are italic.
Example 8: General Sibling Selector
h2 ~ p { color: #e91e63; }
Output: All paragraphs after h2 are pink.
Example 9: :hover Pseudo-class
button:hover { background: #2196f3; color: #fff; }
Output: Button turns blue on hover.
Example 10: :focus Pseudo-class
input:focus { border-color: #ff9800; }
Output: Input border turns orange on focus.
Example 11: :nth-child
li:nth-child(2) { color: #f44336; }
Output: Second list item is red.
Example 12: :not()
button:not(.primary) { background: #eee; }
Output: Buttons without .primary class have gray background.
Example 13: :first-child
p:first-child { font-weight: bold; }
Output: First paragraph is bold.
Example 14: :last-child
li:last-child { text-decoration: underline; }
Output: Last list item is underlined.
Example 15: ::before Pseudo-element
h3::before { content: " "; color: gold; }
Output: Gold star before every h3.
Example 16: ::after Pseudo-element
h3::after { content: " "; color: green; }
Output: Green check after every h3.
Example 17: ::first-letter
p::first-letter { font-size: 2em; color: #2196f3; }
Output: First letter of paragraph is large and blue.
Example 18: ::selection
p::selection { background: #ffeb3b; }
Output: Selected text in paragraph is yellow.
Example 19: :checked
input[type="checkbox"]:checked { accent-color: #4caf50; }
Output: Checked checkboxes are green (modern browsers).
Example 20: :disabled
button:disabled { opacity: 0.5; }
Output: Disabled buttons are semi-transparent.

CSS transitions and animations allow you to smoothly animate changes to properties and create engaging effects using keyframes and transforms.

Example 4: Transition Delay

  from { opacity: 0; }
  to { opacity: 1; }
}

Example 7: Animation Fill Mode

Output: Animation stays at end state.
Example 8: Animation Delay
div { animation-delay: 1s; }
Output: Animation starts after 1 second.
Example 9: Animation Direction
div { animation-direction: alternate; }
Output: Animation reverses every cycle.
Example 10: Animation Iteration Count
div { animation-iteration-count: 3; }
Output: Animation runs 3 times.
Example 11: Transform Scale
div:hover { transform: scale(1.2); }
Output: Element grows on hover.
Example 12: Transform Rotate
div:hover { transform: rotate(15deg); }
Output: Element rotates on hover.
Example 13: Transform Translate

Output: Element moves right on hover.
Example 14: Transform Skew

Output: Element skews on hover.

@keyframes jump {
  to { left: 100px; }

CSS Best Practices & Tips

Follow these best practices to write clean, maintainable, and efficient CSS for your projects.
Tip 1: Use Semantic Class Names
.btn-primary, .card-header, .nav-link
Why: Improves readability and maintainability.
Tip 2: Keep Selectors Short & Specific
.sidebar .menu-item { ... }
Why: Avoids overly complex selectors.
Tip 3: Use CSS Variables for Repeated Values
:root { --main-color: #2196f3; }
Why: Makes theme changes easier.
Tip 4: Group Related Rules
h1, h2, h3 { margin-bottom: 1em; }
Why: Reduces repetition.
Tip 5: Use Shorthand Properties
margin: 10px 20px;
Why: Keeps code concise.
Tip 6: Avoid !important When Possible
/* Only use !important as a last resort */
Why: Makes debugging easier.
Tip 7: Use External Stylesheets
<link rel="stylesheet" href="style.css">
Why: Separates content and presentation.
Tip 8: Minimize Repaints & Reflows
/* Batch DOM changes and use classes for state */
Why: Improves performance.
Tip 9: Use Rems/Ems for Sizing
font-size: 1.2rem;
Why: Supports accessibility and scaling.
Tip 10: Comment Your Code
/* Section: Navigation Styles */
Why: Helps future you and teammates.
Tip 11: Use a CSS Reset or Normalize
@import url('normalize.css');
Why: Ensures consistency across browsers.
Tip 12: Prefer Flexbox/Grid for Layout
display: flex; /* or */ display: grid;
Why: Modern, responsive layouts.
Tip 13: Test on Multiple Devices
/* Use browser dev tools for emulation */
Why: Ensures your site looks good everywhere.
Tip 14: Use BEM or Similar Naming Conventions
.block__element--modifier
Why: Makes large projects manageable.
Tip 15: Avoid Deep Nesting
/* Limit selector depth for performance */
Why: Keeps CSS fast and easy to override.
Tip 16: Use Logical Properties
margin-inline: 1em;
Why: Supports better internationalization.
Tip 17: Prefer Mobile-First CSS
@media (min-width: 600px) { ... }
Why: Easier to scale up for larger screens.
Tip 18: Use Layered CSS for Components
@layer base, components, utilities;
Why: Organizes styles by purpose.
Tip 19: Optimize for Accessibility
/* Use sufficient color contrast and focus styles */
Why: Makes your site usable for everyone.
Tip 20: Keep Learning!
/* CSS evolvesstay updated with new features */
Why: Modern CSS is powerful and always improving.
